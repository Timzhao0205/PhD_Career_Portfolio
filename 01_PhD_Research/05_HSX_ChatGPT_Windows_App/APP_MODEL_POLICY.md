# ChatGPT Windows model and effort policy

## Recommended one-time UI selection

Before pasting the kickoff prompt, select **GPT-5.6 Sol** and **Extra High** reasoning. This keeps
every critical stage on the requested high-end model without requiring later interaction.

## Autonomous budget optimization

If and only if the app exposes per-agent model selection without interrupting the user:

| Work | Preferred model | Effort |
|---|---|---|
| inventory and deterministic extraction | GPT-5.6 Luna | low |
| source discovery and claim organization | GPT-5.6 Terra | medium |
| three-board architecture | GPT-5.6 Sol | high |
| wire-bond/external join | GPT-5.6 Sol | xhigh |
| ceramic package and collision review | GPT-5.6 Sol | xhigh |
| independent red team | GPT-5.6 Sol | xhigh |
| final reconciliation and synthesis | GPT-5.6 Sol | xhigh |

Do not simulate model switching in prose. Log the actual selection. If autonomous selection is not
exposed, retain the parent Sol/xhigh model for the entire mission. Never silently downgrade a
critical stage.
