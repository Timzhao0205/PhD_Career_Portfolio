# CLAUDE.md — Cryogen-Free HTS Component Patent Discovery Engine

> **Research aid — not legal advice.** Generated by an automated research pipeline.
> Patentability, freedom-to-operate, inventorship, and ownership (including Stanford SU-18 /
> sponsor obligations) are fact-specific legal determinations requiring a registered patent
> attorney before any filing, disclosure, or reliance.

This project autonomously discovers, simulates, novelty-checks, and patent-shapes
**cryogen-free / conduction-cooled HTS component** inventions. Read `10_MISSION/MISSION.md`
in full before acting. This file is the always-loaded rule spine.

## THEME (fixed)
Conduction-cooled HTS components cooled by a cryocooler with **limited cooling power**,
**no LN₂ / no cryogen bath**. Anchor exemplar: Hinetics CRUISE-class conduction-cooled
no-insulation (NI) field coils. Scope may extend to HTS cables, MRI/NMR inserts,
accelerator/stellarator coils — but every candidate is scored **primarily on
cryogen-free relevance** (hard multiplier in the rubric).

## HARD RULES (never relax; violations halt the run)
1. **Ownership two-prong gate.** Personal machine/repo/card/home bench only — that
   handles resource-use (prong b) but NOT subject-matter scope (prong a). Any
   candidate whose core reads onto the user's FUNDED lane — HSX plasma diagnostics,
   GaN/AlGaN Hall-effect device development, or battery magnetic imaging — is tagged
   `BLOCKED-PENDING-COUNSEL`, stubbed, and NOT developed. Cryogen-free coil
   manufacturing/thermal work is presumptively outside scope, but the adjudicator
   MUST check each candidate and flag any Hall-sensing adjacency explicitly.
2. **No GaN anywhere.** Not in claims, embodiments, examples, figures.
3. **No "stellarator" in any patent-facing text.** Allowed only as an internal
   application note. Framing is "conduction-cooled / cryogen-free HTS windings."
4. **China ≈ no grace period.** Any enabling disclosure ranks behind priority.
   This engine writes to the private repo ONLY. It NEVER drafts outreach, NEVER
   web-posts artifacts, NEVER emails.
5. **Duty of candor.** All art the engine finds enters the candidate's `IDS_pool`.
   Design-around ≠ concealment. A patent granted to you ≠ freedom to operate.
6. **Number-truth discipline.** Every simulated result is labeled prophetic /
   present tense; NEVER blended with any measured data in one figure/example/table.
   Every quantity carries units + a source or a derivation.
7. **No filings.** The engine proposes; the user and counsel dispose. No document
   is submitted anywhere by the engine.

### FABLE-5 CRITICAL GATES (user-mandated; these three, by my selection)
The three least-reversible, highest-consequence judgments use **Fable 5**:
- **G-PHYS** physics-correctness: is the simulated effect real, not an artifact?
- **G-NOVEL** novelty: is this already patented in US/CN/JP/KR/EU? (the go/no-go)
- **G-CLAIM** claim-survivability: given the art, what core survives + best adjustment?

**Fable-5 down-route caveat (log honestly):** Anthropic safeguards may transparently
route some Fable-5 queries to Opus 4.8; the metrics hook cannot detect this. Each
Fable-gated stage MUST (a) log `models=GATE:fable-5` as *intended*, and (b) note in
`PROGRESS_LOG.md` that the actually-served model is unverifiable from inside the
session and must be confirmed against the external transcript
(`%USERPROFILE%\.claude\projects\`) before a filing-relevant decision relies on it.
A Fable self-report is NOT proof of which model ran.

## RUN-LOG STANDARD (every wave)
Maintain in `80_STATE/`: `RUN_STATE.json` (phase pointer, wave counter, `models`
map, budget used/left, gate results), `PROGRESS_LOG.md` (append-only, one line per
wave, includes `models=…`, effort, timestamps, interruptions/resumes/fallbacks),
`ASSUMPTIONS.md` (A##-numbered autonomous decisions). Run `tools/check_runlog.py`
at every Stop. Purpose: selective batch reruns of any stage on a stronger model.

## PIPELINE (waves; each a resumable checkpoint)
W0 scaffold (haiku) → W1 prior-art harvest (sonnet) → W2 simulation (sonnet) →
W3 Fable gates G-PHYS→G-NOVEL→G-CLAIM (fable-5) → W4 disclosure drafting (opus,
survivors only) → W5 synthesis + run report (opus). Details in `10_MISSION/MISSION.md`.

## SLASH COMMANDS
`/runwave <n>` execute one wave to its checkpoint · `/gates <CFid>` run the three
Fable gates on a candidate · `/status` print RUN_STATE + budget · `/metrics`
summarize model/effort activity from the logs · `/log <note>` append a progress line.
