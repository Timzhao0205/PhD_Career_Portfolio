next round of engineering/simulation effort*, not a filing queue.

## 3. Flagship recommendation

**CF-1 remains the flagship**, consistent with the seed hypothesis. It has the
strongest independent physics verification (the gate hand-derived R_char and confirmed
energy conservation), the clearest mechanism-level differentiation from the nearest
blocking art (coupled single-material zoning vs. Tokamak Energy's discrete
bridges/apertured insulators), and composes with CF-2 (a dependent claim already
references CF-1 co-design) per the original mission's "measure→sense→actuate" family
thesis.

**CF-4 is the strongest second bet** — its underlying physics needed the least
correction of any candidate (the gate's robustness probe, adding a 20% margin reserve
and 2s sensor lag, still showed a 21.5% ramp-time saving with zero violations), and its
blocking prior art is expired, which simplifies both patentability and FTO.

## 4. Budget-phased simulation + prototype plan

All phases reuse the shared conduction-cooled test architecture implied by the CF-1
starter (30-turn NI pancake, cold-head boundary, cryocooler load-line at 20/30/50 K) —
a single 77 K-class conduction rig, per the mission's "shared rig" instruction, can
serve CF-1, CF-2, CF-3, and CF-6 sequentially since they all instrument the same
turn-to-turn thermal network.

**Phase 0 — Sim remediation (no new hardware, ~1–2 weeks equivalent effort each):**
- CF-1: fix Pareto-front rounding (R1), correct `cooling_budget()` units so the
  fixed-cooling-W constraint actually binds (R2), sweep disturbance location for a
  worst-case claim (R3).
- CF-2: fix the same units conflation; add a degraded-cooling scenario and a
  model-error sweep so the telemetry-specific claim element has differential evidence.
- CF-3: **rebuild from scratch** — fix the `min(k[i],k[i+1])` artifact that makes the
  naive baseline inert by construction, and add the electrical bypass network with
  Joule heating that the claimed "current steering" mechanism actually requires. This
  is a precondition for further CF-3 investment, not an optional polish.
- CF-4: close the loop (first-order cold-head thermal ODE + load line instead of an
  exogenous headroom curve), add `alpha<1` margin and explicit sensor-lag modeling.
- CF-6: extend the cooling-budget sweep so the cryogen-free-specific claim has
  evidence (currently the result is invariant to cooling budget, which is backwards).
- CF-7: build the minimal 1-D lead heat-load model G-PHYS said is a precondition for
  any prophetic example.

**Phase 1 — Prior-art closure (parallel with Phase 0, sub-$0 cost, time only):**
- CF-1: native CN/JP/KR patent-portal passes (only Google-Patents mirrors were used);
  pull US12,196,792 and CN115485868A full claims.
- CF-2: **mandatory** — obtain IEEE Xplore #9893361 full text; CNIPA-direct
  verification of CN122194022A (China no-grace-period risk).
- CF-3: **mandatory, highest priority of all follow-ups in this run** — pull
  EP4078630B1's granted claims + INPADOC family. This single document could kill CF-3
  outright or clear it; nothing else in Phase 0/1 matters for CF-3 until this is read.
- CF-6: live KIPRIS/J-PlatPat/CNIPA re-verification of 4 low-confidence ledger entries
  that could demote the candidate to DUPLICATED.
- CF-7: IBM US10,511,168 full-text pull (flagged mandatory by the gate, not yet done).

**Phase 2 — Sub-$1.5K prototype validation (only after Phase 0 sims are fixed and show
real, non-artifactual effects):**
- CF-1: a 6–10 turn NI pancake mockup on a single cryocooler cold-head test stand,
  copper/stainless co-wound zones, measuring both charging tau and a locally injected
  hot-spot ΔT — directly validates or falsifies the Pareto-front claim with real data
  (kept rigorously separate from the simulated figures, per the number-truth hard rule).
- CF-4: a bench ramp-governor demo on the same rig, injecting a synthetic duty-cycling
  cold-head load and comparing static vs. dynamic ramp control in hardware.
- CF-2/CF-3/CF-6: defer prototype spend until Phase 0/1 close — no point building
  hardware for a claim element the simulation doesn't yet support (CF-3) or that a
  single NPL/patent document could invalidate (CF-2, CF-6).
- CF-7: a lead/termination test fixture that runs the combined thermal+electrical
  acceptance sequence once the minimal sim exists to define the acceptance envelope.
