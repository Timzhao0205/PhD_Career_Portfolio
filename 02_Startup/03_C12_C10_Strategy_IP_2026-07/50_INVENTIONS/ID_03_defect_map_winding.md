# ID_03 — Defect-map-synchronized winding

Slot W-03 (21/25, INVENT) · lane C12 · linked U-009, U-001, U-029 · basis PL_P01 WS-P01-2, PL_P04 WH-P04-2 · rev 2, 2026-07-04, per IPRT_03. Drafting aid, not claims.

## 1. Title & field

Defect-map-synchronized winding: winding apparatus plus wind-plan-compiler method/medium that ingest a position-indexed Ic/defect dataset for the specific tape loaded, keep it registered to laid tape using the tape's own mapped defect signatures as fiducials, and dispose each defect against a per-turn Ic(B,θ)-margin model: placement, feed, tension, splice, record. Field: REBCO/NI-HTS coil-manufacturing equipment. CPC: H01F41/06, H01F41/048, H01F41/082, H01F41/094, G05B19/41875.

## 3. Inventive concept

Dataset→plan→execute is old converting art [P-204]; position-indexed roll maps registered across stretch and splice are claimed electrode art [P-206, P-207]; detect-cut-splice escalation during winding is 1993 yarn art [P-208]. The SC-specific residue, found in no fetched reference, is now the claim spine. **(1) Content-based registration:** the cell re-detects, at or upstream of the head, sensed signatures (Ic dip, magnetization anomaly) of defects already in the imported map and re-zeroes position error on them: the payload data is its own fiducial grid. Drift: encoder slip 0.1-0.5% + stretch ~0.05% (30 N, 0.4 mm² CC) → 1.5-5.5 m/km uncorrected vs ±0.5 m guard windows. **(2) Margin-referenced placement:** a stored per-turn Ic(B,θ) load model of the coil decides each feature's disposition (placement shift, derate, splice-out when no admissible landing exists), not bare presence/severity; as-laid inventory and actions feed a serialized per-coil record. Live during-winding electrical sensing stays in ID_01.

## 4. Difference vs nearest prior art (fetched/re-fetched 2026-07-04)

## 5. Claim sketch (plain language — drafting aid, not claims)

Primary independent stays **apparatus**: visible in a sold machine, single actor, distanced from the method art. Second independent (method + CRM) per IPRT_03: compiling completes the method, one actor; the CRM form reaches a tape vendor giving planner software away. Operator-flag limb struck: read on inspect-and-flag winders [P-205].

1. (independent, apparatus) A superconducting-tape winder (payoff, former drive, tension actuator, length tracking) whose controller (a) holds a position-indexed conductor-quality dataset for the specific tape loaded, from producer characterization and/or a scanner of the apparatus; (b) keeps dataset-to-tape registration by re-detecting, with a sensor at or upstream of the head, the signature of at least one mapped defect, correcting accumulated length error on it; (c) holds a stored position-dependent margin model of the coil (per-turn field magnitude/angle → local Ic margin); for a mapped feature it selects and automatically executes, against that model, at least one of: shifting the wind plan so the feature lands where computed margin absorbs it; modulating feed or tension in a guard window; initiating splice-out when no admissible landing exists.
2. Registration estimator fuses encoder counts with a tension-dependent stretch model; re-zeroed also at printed marks and splices; ≤ ±5 cm/km.
3. Margin tiers (Ic < 70% nominal → splice-out; 70-90% → shift/derate; > 90% → log-only) inside guard windows.
4. Pre-wind planner computes each landing (turn, angle, layer) from former geometry; picks start offset, splice positions, pancake assignment minimizing stored margin penalty.
5. As-laid defect inventory (coil coordinates), actions, predicted-performance figure written to a serialized per-coil record (feeds ID_07).
6. (independent, method + CRM) On one computing device: receive the position-indexed dataset for a specific tape length; compute each feature's landing from stored former geometry; evaluate each landing against a stored Ic(B,θ)-margin model; select start offset, splice list, feed/tension schedule so features land where margin absorbs them; emit a machine-executable wind plan. Winder execution is the plan's defined effect, not a claimed step (single actor).

Dependents 2-5 are severable; the set survives deleting any one; 3-4 also hang from claim 6.
