# ID_09 — Multi-modal quench-detection instrument: any-modality arm/confirm fusion with self-measured rated trip latency

Slot W-13 (21/25, INVENT) · Rev 2, 2026-07-04 · U-022, U-023, U-026 · PL_P09 WS-P09-1 / DD_C11 §2 / IPRT_09 · Drafting aid, not claims.

## 1. Title & field

Coil-agnostic rack instrument protecting a single HTS/NI/PI coil: voltage-tap bridge fused with ≥1 sensor class of different physics — external Hall-array current-redistribution sensing, optional external absolute-referenced fiber channel — in which ANY modality can arm the trip and a different modality must confirm, behind a hardware-enforced, continuously self-measured trip-path latency, driving a deterministic ms-class trip output. Field: SC-magnet protection instrumentation (CPC H02H7/001, H01F6/02; G01K11/32 adjacency). Superconducting coils only — no transformer condition monitoring in any embodiment, example, or claim language (Magnefy wall).

## 2. Problem & demand evidence

No commercial catalog QD instrument for HTS magnets exists; every program self-builds or tenders bespoke — Fermilab's 3-tier Mu2e stack, KIT UniQD institutional-only, Startorus filing its own QD patents (U-022; GP retry 2026-07-04 still shows nothing indexed). The detection→dump chain has no latency owner: ITER spreads 3,000 sensors over 20+ tenders (~EUR 25M) inside a 2-3 s budget, and its bespoke architecture is heterogeneous defense-in-depth — proof serious programs need exactly this and nobody sells it (U-023; [NPL-09-1/-2]). Fiber channels are DIY — general-purpose interrogators, uncalibrated at cryo, no trigger-rated output, lead vendor Luna in PE churn (U-026). Physics: HTS normal zones propagate cm/s-class, so taps fire late; in NI/PI coils radial bypass suppresses tap voltage exactly while danger grows (31.4 T LNI post-mortem, DD_C11 §2) — the failure mode a tap-armed detector never sees. Cryomagnetics bundles LTS-class QD free, so the instrument must do what bundled logic cannot: arm on the NI signature itself and certify its own speed. MPMU window $20-80K/system; the C10 strategy absorbs this as the protection attach (BASELINE §B).

## 3. Inventive concept (what is new)

Heterogeneous-latency QD exists at facility scale — ITER runs a voltage primary, thermo-hydraulic secondary, and pressure tertiary on redundant FPGA pairs [NPL-09-1]; EAST runs CWW plus model-based secondary [NPL-09-5] — but always as bespoke, facility-tuned engineering with fixed channel roles and paper latency budgets. What is new is an instrument that makes the fusion role-symmetric and self-certifying. (i) **Any-modality arming with per-modality confirmation rules:** a threshold event on ANY input channel arms the trip; a channel of different physics must confirm within a bounded window chosen by the arming modality's stored rule. Fast-arms-slow-confirms rejects flux-jump/ramp/pickup false trips; slow-arms-fast-checks captures the tap-silent NI event — the azimuthal Hall asymmetry of current-redistribution onset arms, and the tap channel confirms by a stored look-back over its record for sub-threshold resistive growth it would never have tripped on alone. Unconfirmed arms release and log. (ii) **Self-measured rated latency:** every channel carries an injection test port (pilot tone into the tap network; stimulus conductor at the Hall elements; optical loopback); the instrument periodically injects, measures each channel's response latency and integrity plus the end-to-end arm-confirm-trip path through its hardware-timed logic, publishes the measured figures as a machine-readable rated-latency status, and inhibits/derates rated trip operation whenever any self-test is stale or failed. Confirmation windows are keyed to measured latencies, not factory constants. All coil-agnostic and terminal/external: single coil, per-coil stored parameters, no inter-coil comparison, no in-winding sensors (§4). No fetched patent family claims ≥2 heterogeneous sensor classes fused in a QD instrument (PL_P09 Q1 bounded absence — patents only; deployed NPL practice is acknowledged and distinguished in §4).

## 5. Claim sketch (plain language; drafting aid, not claims)

**Independent: apparatus.** Both live TE threats are methods practiced inside magnets (an instrument omits their elements structurally); infringement attaches to the vendor at sale/import; every element is observable on one purchased unit (§9); CN utility-model dual-filing requires apparatus.

1. A quench-detection instrument for a superconducting coil, comprising: (a) a first input channel receiving voltage-tap signals of a single coil through a compensation/bridge network yielding a resistive-voltage estimate at a first response latency; (b) at least one second input channel receiving signals from sensing elements external to the coil winding, responsive to a physical quantity other than tap voltage, with a different response latency; (c) fusion logic in which a threshold event on ANY input channel arms a trip decision, the trip asserted only on confirmation by a channel of a different physical principle within a bounded window selected, per the arming channel, from a stored per-modality rule set that includes both (i) a faster-channel arm confirmed by a slower channel within a window sized to the slower channel's measured latency and (ii) a slower-channel arm checked against the faster channel's record over a stored look-back interval, an unconfirmed arm being released and logged; (d) a self-test arrangement comprising an injection test port for each input channel (pilot tone / stimulus-conductor pulse / optical loopback per modality) through which a stimulus is periodically injected and that channel's response latency and integrity measured, together with hardware-timed logic measuring end-to-end arm-to-trip path latency; (e) a rated-latency status output publishing the measured per-channel and end-to-end latencies as a machine-readable health/rating signal, asserted only while every measured latency fits a stored budget and every self-test result is fresh, the instrument inhibiting or derating rated trip operation per a declared degraded-mode rule when any self-test is stale or failed; (f) a trip output circuit issuing the de-energization/dump command within the budget, and a supervisory output asserted on any self-test failure; (g) a per-coil parameter store from which thresholds, per-modality windows, look-back depths, staleness limits, and the budget are loaded.

Dependents (each from claim 1 only; each survives removal of any other):
2. Second channel comprising plural Hall elements azimuthally distributed about a no-insulation/partial-insulation coil; a spatial-asymmetry (harmonic) measure of the external field indicates current-redistribution onset and ARMS on tap-quiet bypass events; the look-back check searches the tap record for sub-threshold resistive growth; the confirmation window is derived from the coil's stored characteristic time (measured L/Rc).
3. Second channel comprising an external optical-fiber temperature input evaluated against a stored per-fiber absolute cryogenic calibration, never fiber-to-fiber differential.
4. Confirmation windows and channel arming/confirming roles automatically re-derived from the self-test-measured latencies each test interval (latency-measurement-driven fusion reconfiguration), each re-derivation logged.
5. Degraded mode: on a confirming-channel self-test failure, trip on a surviving channel alone at a tightened threshold with the supervisory output asserted (declared fail-operational behavior).
6. Trip output comprising redundant, diverse paths (force-guided electromechanical and solid-state), both monitored, asserted together.
7. Per-coil parameter store loaded from a machine-readable coil manufacturing record binding thresholds and windows to the coil serial (W-19/ID_07 seam).
