# IPRT_07 — Adversarial IP review of ID_07 (per-coil digital birth certificate)

## 1. Closer prior art the designer missed

ID_07 §4 benchmarked only against SC art plus one generic MES patent. The record/passport layer is a crowded cross-industry field the search never entered:

- **US11151512B2 (Boeing, priority 2018-12-14, Active).** Per-part history blockchains interlocked with vehicle-configuration blockchains; hash-linked records locked while the part is installed. Per-unit hash-chained manufacturing/usage records bound to part identity, granted, pre-dating ID_07 by 8 years.
- **US12154078B2 (Hochman, priority 2019-03-01, Active).** Aircraft part documentation with universal tracking number, blockchain module, media capture bound to the ID — the specification uses the phrase "digital birth certificate." The title concept itself is anticipated vocabulary.
- **CN111832676A (江苏英耐杰新能源, filed 2020-07-02, deemed withdrawn 2024 — prior art only).** Barcode-bound per-cell battery traceability: each cell record = sequence code + OCV + internal resistance + timestamp, appended per production stage (cell test → module → PACK), uploaded to a server. Structurally this is claim 1 minus the SC physics: per-unit records built as production events occur, bound to a machine-readable identifier.
- **US20210084497A1 (Siemens, priority 2018-02-20, abandoned — prior art).** Control unit for tamper-proof capture of live monitoring data: hash-of-preceding-record chaining, attestation, secure logging of sensor streams. Dependent claim 3 (hash chain over stream digests, signed) is dead on arrival as an independent contribution.
- **NPL:** EU Reg. 2023/1542 (OJ 2023-07-28) mandates per-battery passports with QR "printed or engraved on batteries" — regulation-published architecture for exactly this record-bound-to-unit pattern. Worse for the "no SC traceability" premise: CERN's MTF (EPAC 2002) has issued per-magnet electronic manufacturing/test folders — manufacturer data, procedures, test results, NCRs entered during manufacture at supplier sites for LHC SC magnets — since 2002; Devred et al., SUST 27:044001 (2014) documents the ITER conductor database: per-lot/unit-length IDs, ~27,000 Ic measurements, 7 control points, 20 suppliers. PL_P04 WH-4's "no SC-specific traceability" holds only at claim level; as 103 context the concept is 20+ years old inside this exact industry.

## 2. Obviousness attack

Strongest combination: known winding cell with sensors (Canon [P-079]-adjacent art) + US7854057B2 [P-062] spool dataset whose *claimed purpose* is positional use by the coil customer + CN111832676A per-station event-appended unit records + Siemens '497 tamper-evident stream capture + generic MES [P-079]. Every claim-1 element performs its known function; the spool-meters→(turn, angle, layer) transform is arithmetic on logged kinematics — the obvious bookkeeping step once you consume the P-062 map as intended. The remaining "weight" is *what data the record contains* (predicted Rc, joint inventory): printed-matter risk — content earns weight only via functional relationship, and ID_07 *disclaims the functional consumer* (claim 6 boundary pushes acting-on-the-record to ID_14), sawing off its own §101/printed-matter branch. Device-binding honesty: the winder/sensors are conventional; the novelty locus is software on an SBC — Alice risk in US, algorithm-feature screening in CN (PL_P10: method-only Injet rejected; hardware anchors here are thin).

## 3. Enablement / budget realism

Element (c) needs a real producer defect/Ic map for the identified spool — no BOM line buys a mapped commercial spool; a self-synthesized map demonstrates schema, not the claimed input. ## 4. Contamination probe

## 5. Design-around

Post-hoc MES assembly of a *conforming open-schema* certificate escapes live-append and is invisible at the interface — the buyer cannot audit binding time. The claim does not protect the 2029 wedge; the calibration corpus (already trade-secret) and open schema do.

## 6. Strategic value

Genuinely new residue: the spool→coil kinematic transform and in-fixture joint event append — worth dependents inside ID_01/ID_03's apparatus filings (whose §10 already floats a combined provisional), not a 13th-ranked standalone family. The schema should be open (the disclosure agrees); the hash layer is genericized.
