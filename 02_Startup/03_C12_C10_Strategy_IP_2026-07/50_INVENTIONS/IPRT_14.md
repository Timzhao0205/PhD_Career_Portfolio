# IPRT_14 — Adversarial IP review of ID_14 (coil-record-driven converter/protection parameterization)

Red-team run 2026-07-04. All new references fetched this run (fragments below). Verdict scale per IP_GROUND_RULES.

## 1. Closer prior art the designer missed

ID_14 §4 stops at one TEDS primitive [P-171]. "Device record configures driver, protection limits included" is a decades-old universal pattern in three industries the search never entered:

- **US6211639B1 (Bosch, priority 1997-08-08, displays Expired-Fee).** Drive device regulating a servomotor; memory IN THE MOTOR HOUSING stores motor-specific data; the control module configures its regulating device from it. Spec stores "maximum allowable rotation speed, maximum allowable current" limit values and control parameters: "the motor current supply with the respective current regulation can be automatically adjusted to a motor after reading the motor characteristics." That is claim 1(a)-(d) for a coil-plus-converter system, 1997.
- **US5572110A (Intel, priority 1994-12-15, displays Expired-Lifetime).** Smart battery's stored charging characteristics feed a logic unit deriving desired charging current AND voltage; the charger power stage executes them. Record-derived charge-profile enforcement, 1994.
- **USRE36495E (Picker, priority 1994-08-05, displays Expired-Fee).** MR coils carry an electronic ID; the system interrogates it, retrieves coil-specific parameters/modes, and blocks "operation of defective coils and operation of coils in an inappropriate mode" — dependent 7's verified-identity unlock, from the assignee whose US6127826A the designer did cite.
- **NPL:** HEIDENHAIN EnDat OEM memory ships the motor's "electronic ID label... motor model, maximum current rating" in the encoder; Dewesoft on TEDS: the DAQ "sets the channel up completely automatically, applying the correct scaling and engineering units, signal conditioner gain." Commodity practice, not invention.

## 2. Obviousness attack

Strongest 103: US6211639B1 (record-in-device configures drive regulation and current limits) + FSU [P-028] (motivation: Rc and L are THE NI charging-law parameters) + CERN MTF / ITER conductor DB (IPRT_07 NPL: per-magnet manufacturing records are 20-year SC practice) = claim 1. The drafting invites it: "at least one of... a winding-parameter log" lets a generic winding log satisfy (b), and "at least one operating or protection parameter" is intended-result wording — Bosch reads on that branch with only the SC preamble as residue. ID_14 does hold the functional relationship IPRT_07 pushed onto it (printed-matter survivable), but only where a derivation is recited algorithmically — and every SC-specific derivation (τ from Rc distribution, joint-sum trip, defect derate, dump partition) sits in dependents. Dep 2 is Hahn-2011 arithmetic; dep 3 automates the commissioning engineer's Ohm's-law margin; dep 6 (nameplate + model prediction + runtime-log drift alarm) walks into Siemens US10762475B2's ACTIVE fleet-twin fence (~2037, PL_P10 §2) — never charted.

## 3. Enablement / budget realism

$2.9K credibly demos deps 2-4, 6-7 on the inherited ID_01/ID_07 rig (three-disclosure stack; upstream slip strands it). The demo proves the elements Bosch renders obvious and skips the element carrying the SC-specific weight.

## 4. Contamination probe

No-field-sensor construction is genuinely good wall hygiene; lanes hold; converter firmware sits outside the stated SOW. Two propagation risks: (i) E1 consumes ID_01/ID_07 outputs, so their wall failures inherit; (ii) IPRT_07 ordered defensive publication of the ID_07 schema right after its host provisional — if that text describes consumption/derivation semantics before ID_14 files, CN novelty dies (no grace period). Sequencing conflict unaddressed.

## 5. Design-around

(a) Operator keys three numbers from the paper test report into a stock supply UI: no record interface, human derivation — clean escape; dep 7 catches nobody who never claimed certificates. (b) Worse, the portfolio self-contradicts: §11(b) says in-situ identification "needs a field sensor and walks into [P-028]" — sibling ID_11 exists precisely because terminal-only impedance probing needs neither. A low-current commissioning pre-ramp extracts τ and effective resistance at the terminals and auto-populates the same envelope: no record, no FSU fence, no ID_14. The un-designaroundable residue is data invisible at the terminals and needed before first cooldown — per-joint breakdown, coordinate defect positions. The sketch never anchors there.

## 6. Strategic value

Not vanity, not ID_07 in disguise: with IPRT_07 killing the standalone certificate family, ID_14 is the only vehicle left carrying the interface claim; it sits on the actual 2029 wedge product (coil-aware supplies for U-025 MJ-class owners), and the CN seam is dead-art-uncontested (PL_P10 WH-5). Properly narrowed, it forces incumbents to choose blind supplies or cold-time identification runs. As sketched, it grants nothing enforceable.

## Verdict: REWORK

Exactly: (1) rewrite claim 1 — delete the "at least one of" menu (the winding-log branch is Bosch), require Rc data PLUS a spatially resolved field (per-joint inventory or coordinate defect map), recite two derivations algorithmically (ramp trajectory bounded by record L/Rc lag; trip threshold from summed per-joint resistances), both enforced; (2) fence the identification around by claiming terminal-invisible, pre-cooldown data; rewrite §11(b) to stop contradicting ID_11; (3) chart US6211639B1 / US5572110A / USRE36495E in §4; chart or drop dep 6 against US10762475B2; (4) mark dep 5 paper-supported or add a high-inductance coil; correct the kJ-class BOM line; (5) file ID_14's provisional BEFORE any ID_07 schema publication and strip consumption semantics from the published schema.
