- **Primary (buyer/spec):** Atlas L02-043, NVIDIA, "NVIDIA 800 VDC Architecture Will Power the Next
  Generation of AI Factories" (2025, `buyer_procurement`, T1) -- NVIDIA's reference architecture is the de
  facto spec every rack power vendor builds to. Refreshed via new source **P3R2-C-02-S01** (NVIDIA,
  "Building the 800 VDC Ecosystem," Oct 2025, fetched): "high-power capacitors and supercapacitors are
  placed close to the compute racks" to "absorb the high-frequency power spikes and fill the brief valleys
  created by LLM workload idle periods" -- an explicit, dated (2025) restatement of the exact buffer function
  C-02 proposes to productize, plus named ecosystem partners (ABB, Eaton, GE Vernova, Hitachi Energy,
  Schneider, Siemens, Vertiv, Delta).
- **Independent second:** Atlas L02-044, Data Center Dynamics, "OCP Members Tout DC Power in the Data
  Center..." (2026, `trade_press`, T2) -- reports Google/Meta/Microsoft co-authoring the OCP Mount Diablo
  working group with vendor partners Eaton/ABB/Siemens; confirms (fetched this session) the working group
  "plans to release an updated white paper later in 2026," i.e., active, ongoing spec development, not a
  static 2025 snapshot.
- **Verdict: demand is real.** GPU pulsed-load transients and DC-link reliability are named, current
  (2025-2026), buyer-side problems, not a hypothetical. The risk is not "does the pain exist" but "will the
  buyer-side spec absorb the function before a startup can sell into it" (see Section 5/6).

## 3. Price signals and bottom-up arithmetic

No vendor discloses a $/kW price for an "AI-rack pulsed-load buffer module" as a category -- it does not
yet exist as a catalogued SKU (consistent with the idea's own framing). Bottom-up arithmetic below, with
every input labeled:

- Addressable racks: NVIDIA states GPU rack power is moving toward "a megawatt per rack" (S01, fetched
  fact). **ASSUMPTION:** total AI-accelerator racks shipped globally 2028-2030 = 150,000-400,000/yr (wide
  range; no fetched shipment-count source this session -- treat as an unverified planning assumption, not a
  claim).
- Buffer attach rate: **ASSUMPTION** 20-50% of new 800V racks specify an external buffer module rather than
  relying solely on BBU/capacitor-shelf integration by the rack OEM (uncertainty driven directly by the
  idea's own commoditization kill trigger -- if OCP/NVIDIA bundle the function, attach rate for a
  standalone vendor collapses toward 0%).
- Module price: **ASSUMPTION** $3,000-$8,000 per 30-100kW shelf module (derived by analogy to Vicor's
  disclosed $/W-class dense-module pricing tier, S02, not an official price list -- Vicor does not publish
  list prices; this is an inference, flagged low-confidence).
- Resulting addressable revenue range: 150,000 racks x 20% attach x $3,000 = **$90M/yr** (low case) to
  400,000 racks x 50% attach x $8,000 = **$1.6B/yr** (high case). This ten-to-twenty-times spread reflects
  genuine uncertainty in shipment volume, attach rate, and price -- not a resolved TAM. Film-vs-electrolytic
  capacitor cost multiplier (~3-4x per unit capacitance, from general industry technical explainers, not
  independently fetched this session) is an additional unverified input to the bill-of-materials case and
  should be re-derived from a real BOM before capital commitment.

## 4. Certification and export constraints

- No dedicated safety/certification standard specific to "rack-level pulsed-load buffer modules" was found
  this session; the nearest applicable framework is general power-conversion equipment safety/EMC standards
  (e.g., IEC 62477-class), not independently verified here -- gap flagged for P5/P6.
- Export control: GaN/SiC discrete power devices were not found to carry a dedicated new 2026 ECCN beyond
  the general EAR framework; the controls identified this session target advanced-compute chips (ECCN
  3A090/4A090) and semiconductor manufacturing equipment (e.g., MOCVD under 3B001.a.2), not industrial
  GaN/SiC power-conversion parts. **This is a web-search-only finding, not independently fetched from BIS
  primary regulatory text this session -- treat as an unverified assumption and re-check the current EAR
  Commerce Control List directly before finalizing the dual-entity manufacturing plan.** If it holds, the
  idea's claimed "good" export separability (buffer module manufacturable with local devices in both a US
  and a CN entity) is plausible.
